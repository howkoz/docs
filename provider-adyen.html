1
Create Session
POST /sessions with amount, currency, returnUrl, and recurringProcessingModel: Subscription. Adyen returns a sessionId and sessionData. No card data sent at this step — all handled client-side by Drop-in.
POST /sessions → sessionId + sessionData
2
Drop-in Renders
Client loads Adyen Drop-in UI with sessionId. Cardholder enters card details. Drop-in handles device fingerprinting silently — collects browser/device data required for 3DS2 AReq.
AdyenCheckout({ session: sessionId })
3
3DS2 Authentication
Adyen sends AReq to issuer with 100+ data elements. Frictionless: issuer returns ARes Y — payment proceeds without cardholder action. Challenge: Drop-in renders OTP/biometric inline. Regulatory mandate active in EU/UK from 2021.
AReq → frictionless (ARes:Y) | challenge
4
Token Stored
On authorised result, Adyen returns recurringDetailReference in the payment result. Unicity stores this as the subscription token. Used for all subsequent MIT charges — never requires cardholder interaction again.
recurringDetailReference → store in vault

Retrieve Token

Webhook Result
AUTHORISATION notification fires to Unicity webhook. Check success: true. Log resultCode, reason, and any refusalReason codes for analytics. Always verify via GET /payments/{pspReference} for authoritative result.

Retry / UCOF on Failure
Soft decline: retry Day +3, +5, +7 using ContAuth. If card expired or amount changes on retry: switch to shopperInteraction: ContAuth + recurringProcessingModel: UnscheduledCardOnFile. Preserves liability exemption on retries.
UCOF for dunning retries

Poland — KEY DIFFERENTIATOR
BLIK
Poland's dominant payment method — 75%+ of Polish e-commerce. Mobile-code-based authorization. Adyen supports BLIK natively, including recurring BLIK (mandate-based). This is the critical advantage over Nexi, which requires a separate Netaxept integration for BLIK in Poland.
PLNNative on AdyenRecurring supportRequires Netaxept on Nexi
Netherlands
iDEAL → SEPA
iDEAL is a one-time bank redirect. Adyen converts iDEAL to SEPA Direct Debit for recurring — single mandate covers all future charges. Correct approach for NL autoship subscribers.
EURSEPA mandate flowOne-time then recurring
Europe-wide
SEPA Direct Debit
Bank-to-bank EUR debit with mandate. Covers all SEPA-zone countries. 3-5 day settlement. Return risk (SEPA returns up to 8 weeks). Suitable for subscriptions where card on file isn't available.
EURMandate-based recurring3-5d settlement

Denmark's national debit scheme. Required for DKK market coverage. Adyen supports Dankort natively through local acquiring in Denmark. Recurring support available.
