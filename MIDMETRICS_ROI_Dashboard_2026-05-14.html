Methodology, Currency, Caveats

Period

2025-05-01 to 2026-05-14 (~12.5 months).

Order-level deduplication (per Processing Rate Methodology Brief)

Each input row is one ATTEMPT. We dedupe on the order ID per provider. An order counts as APPROVED if ANY attempt on that order succeeded. Approval source per processor:

  • Nuvei USA: NUVEI-US-MAY2025-MAY2026.csv - dedup on orderid
  • Nuvei Canada: existing Accepts CSV (Canada-9766 MID only) - dedup on ExternalOrderId
  • Trust: TRUST-TRANSACTIONS file - dedup on Order Reference
  • Worldpay: Authorizations file - dedup on Order ID

Data sources (in-period totals)

  • Midmetrics invoices: 54 | $67,115 USD
  • RDR alerts: 1,174 | 1,172 resolved | $300,029 USD
  • Ethoca/Verifi alerts: 1,361 | 1,299 resolved | $489,067 USD refunded
  • Nuvei USA CBs: 1,114
  • Nuvei Canada CBs: 62
  • Trust CBs: 355
  • Worldpay CBs: 340 (1st chargebacks)
  • Nuvei USA approved orders: 404,345
  • Nuvei Canada approved orders: 164,366
  • Trust approved orders: 63,405
  • Worldpay approved orders: 325,447

Definitions

  • CB ratio: chargebacks / approved orders (deduped) - real measured number
  • Capture rate: alerts / (alerts + chargebacks) - real, % of disputes caught first
  • True prevention: RDR resolved + Ethoca/Verifi 'Resolved' family of resolution reasons
  • Redundant alert: Ethoca/Verifi resolved with reason indicating customer was already taken care of

What this dashboard does NOT include

  • Per-CB chargeback fees (would require parsing Nuvei monthly statement PDFs).
  • Synthetic / projected ratios. Removed in V4 to avoid model assumptions.

Caveats

  • Worldpay CB count uses '1st chargebacks count' (excludes additional disputes + re-presentments).
  • Worldpay attempts span Apr 2025-May 2026 in source; we filter to in-period.
  • Nuvei USA approval source switched in V5 from streamed Hydra Accepts to direct Nuvei export (more comprehensive).
